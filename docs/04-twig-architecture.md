# Архитектура Twig

Проект использует шаблонизатор **Twig**.  
Все HTML-страницы генерируются автоматически из Twig-шаблонов.

⚠ Важно:  
HTML в папке `source/` не редактируется вручную.  
Все изменения вносятся только в Twig.

---

## Общая схема

```
source/layout/        # Twig-шаблоны
        ↓
gulp twig
        ↓
source/*.html         # сгенерированные HTML
```

---

## Типы страниц

В проекте есть 3 типа Twig-страниц:

### 1. Основные страницы сайта

```
source/layout/*.twig
```

Примеры:
- главная
- услуги
- контакты
- прочие страницы

Собираются командой:

```
gulp twig
```

---

### 2. Статьи услуг (Articles)

```
source/layout/articles/**/*.twig
```

Это страницы, на которые ведут блоки услуг на главной.

URL после сборки:

```
source/articles/...
```

Собираются командой:

```
gulp twig-articles
```

---

### 3. Статьи блога (Blog)

```
source/layout/blog/**/*.twig
```

Это отдельные статьи раздела «Блог».

URL после сборки:

```
source/blog/...
```

Собираются командой:

```
gulp twig-blog
```

---

## Компоненты

Повторно используемые части сайта лежат в:

```
source/layout/components/
```

Примеры:
- карточки
- блоки
- секции
- элементы интерфейса

Подключаются через Twig `include`.

---

## SVG-иконки

SVG-иконки собираются в Twig-спрайт:

```
source/layout/partials/_icons.twig
```

Он подключается в базовый шаблон и используется на всех страницах.

---

## Куда вносить изменения

| Что нужно изменить | Где править |
|-------------------|------------|
Текст на странице | соответствующий `.twig` |
Статья услуги | `layout/articles` |
Статья блога | `layout/blog` |
Повторяющийся блок | `layout/components` |
Иконки | `assets/images/icons` |

---

## Важно помнить

- HTML генерируется автоматически
- Twig — единственный источник верстки
- После изменений запускать `gulp`
